This document outlines all identified UI/UX issues across the Secure Exam Browser

1. ACCESSIBILITY ISSUES
================================================================================

1.1 Poor Color Contrast
    - Light gray text on dark backgrounds may not meet WCAG AA standards
    - Examples: text-slate-400 on dark backgrounds throughout the application
    - Impact: Users with visual impairments struggle to read content
    - Affected Files: launch.html, exam.html, submission.html, dashboard.html

1.2 Missing ARIA Labels
    - Buttons lack accessible descriptions for screen readers
    - Examples: Settings button, Notifications button, Flag button
    - Impact: Screen reader users cannot understand button purposes
    - Affected Files: exam.html, dashboard.html, verification.html

1.3 No Visible Focus Indicators
    - Interactive elements lack clear focus states for keyboard navigation
    - Impact: Keyboard-only users cannot see which element is focused
    - Affected Files: All HTML files

1.4 Font Size Too Small
    - Many text elements use text-xs which is very small (12px)
    - Examples: Status labels, timestamps, secondary information
    - Impact: Difficult for users with low vision to read
    - Affected Files: launch.html, exam.html, submission.html

================================================================================
2. MOBILE RESPONSIVENESS ISSUES
================================================================================

2.1 Exam Sidebar Not Collapsible

    - Affected File: exam.html

2.3 Header Information Overflow
